# Contributing to 🧵 Naples 🇮🇹

Thank you for your interest in contributing to this project!

## Getting Started

1. Fork the repository.
2. Clone your fork:

```bash
git clone https://github.com/PsHyCo71/Naples.git
```

3. Navigate to the project directory:

```bash
cd Naples
```

4. Create a new branch for your changes:

```bash
git checkout -b feature/your-feature-name
```

## Development Guidelines

- Write clean, readable, and maintainable code.
- Follow the existing coding style and project structure.
- Keep changes focused on a single issue or feature whenever possible.
- Add comments only when they improve code readability.
- Update documentation when necessary.

## Commit Messages

Use clear and descriptive commit messages.

Examples:

```text
feat: add task filtering by date
fix: resolve schedule loading issue
docs: update installation instructions
refactor: simplify task management logic
```

## Pull Requests

Before submitting a Pull Request, please ensure that:

- Your code builds successfully.
- All tests pass.
- Documentation has been updated if applicable.
- Changes have been reviewed for quality and consistency.

### Pull Request Process

1. Push your branch to your fork.
2. Open a Pull Request against the main branch.
3. Provide a clear description of the changes.
4. Link any related issues if applicable.

## Reporting Issues

When reporting a bug, please include:

- A clear description of the issue.
- Steps to reproduce it.
- Expected behavior.
- Actual behavior.
- Screenshots or logs (if relevant).

## Feature Requests

Feature suggestions are welcome. Please provide:

- A clear description of the proposed feature.
- The problem it solves.
- Any alternative solutions you have considered.

## Code of Conduct

Please be respectful and constructive when interacting with other contributors.

## Questions

If you have any questions, feel free to open an issue or start a discussion.

Thank you for contributing!
